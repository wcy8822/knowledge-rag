## 🔍 核心功能特性

### 1. 智能文档处理
- **Excel专项策略**:
  - 主键识别: 自动识别ID字段
  - 行聚合: 按行生成结构化句子
  - 元数据: sheet_name/row_range/primary_key

- **PPT专项策略**:
  - 幻灯片粒度: 每页独立处理
  - 标题+正文+备注合并
  - 智能二次切分

- **代码专项策略**:
  - 函数/类级别解析
  - 结构化摘要: 职责/参数/返回/异常/依赖

### 2. 混合检索引擎
- **向量检索**: BGE-M3本地嵌入，语义相似度
- **BM25检索**: 传统关键词搜索，精确匹配
- **智能融合**: 0.6向量 + 0.4BM25权重

### 3. 企业级安全

- **最小暴露**: 问答只发送Top-N片段
- **审计日志**: 完整操作记录，可追溯

### 4. 生产级监控
- **实时指标**: Hit@K、nDCG、延迟、空集率

## 🚀 部署和使用

### 快速启动
```bash
# 1. 环境初始化
chmod +x scripts/*.sh
./scripts/setup.sh

# 2. 文档摄入
./scripts/ingest.sh --input-dir ./data/uploads

# 3. 启动服务
./scripts/serve.sh

# 4. 访问API
curl http://localhost:8000/health
```

### API接口
- **搜索**: `POST /api/v1/search`
- **问答**: `POST /api/v1/search/ask`
- **摄入**: `POST /api/v1/ingest/files`
- **管理**: `GET /api/v1/admin/system`
- **指标**: `GET /api/v1/metrics`

## 🔧 技术决策说明

### 1. 技术选型理由
- **BGE-M3**: 中文嵌入效果最佳，开源可商用
- **ChromaDB**: 轻量级向量数据库，易部署
- **FastAPI**: 现代化Python框架，自动文档生成
- **SQLite**: 轻量级元数据存储，无需额外服务

### 2. 架构设计原则
- **模块化**: 高内聚低耦合，易于扩展和维护
- **可配置**: 所有关键参数可配置，适配不同环境
- **可观测**: 全链路监控，问题可定位
- **安全优先**: 本地优先，数据安全可控

### 3. 性能优化策略
- **批处理**: 嵌入和检索批处理，提高吞吐
- **缓存机制**: 查询结果缓存，减少重复计算
- **异步处理**: I/O操作异步，提高并发
- **资源复用**: 模型和服务实例复用，减少启动开销
