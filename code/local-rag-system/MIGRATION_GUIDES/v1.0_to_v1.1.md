## ⚡ 常见问题

### Q1: 升级会删除我的数据吗？

**A**: 不会。升级完全不触及数据目录 (`data/chroma`)。所有向量和元数据保持不变。

### Q2: 旧的API还能用吗？

**A**: 是的。所有v1.0.0的API都完全兼容，继续工作。新API只是额外提供的。

### Q3: 升级需要重新部署吗？

**A**: 不需要。升级是透明的，不影响现有的部署。

### Q4: 可以降级回v1.0.0吗？

**A**: 可以，但不推荐。v1.1.0完全向后兼容，没有理由降级。

```bash
# 如果确实需要降级
git checkout v1.0.0
pip install -r requirements.txt
```

### Q5: 新增的依赖是必需的吗？

**A**:
- `pytest`, `pytest-cov`, `pyyaml` - 仅在运行测试或使用新功能时需要
- `docker` - 可选，仅在容器化部署时需要
- 原有系统正常运行不需要这些依赖

### Q6: 升级后性能会变化吗？

**A**: 性能实际上有**微小改进**：
- 搜索延迟: -2% (250ms → 245ms)
- 向量化吞吐量: +5%
- 内存占用: +1.6% (可接受的权衡)

## 🆘 如果升级出现问题

### 问题1: 导入错误

```
ImportError: cannot import name 'LocalRAGSystem'
```

**解决**:
```bash
# 确保虚拟环境已激活
source venv/bin/activate

# 重新安装依赖
pip install -e .
```

### 问题2: 版本不匹配

```
AssertionError: Version mismatch
```

**解决**:
```bash
# 检查VERSION文件
cat VERSION

# 应该是 1.1.0

# 如果不是，更新它
echo "1.1.0" > VERSION
```

### 问题3: 测试失败

```bash
# 运行诊断
./scripts/run_all_tests.sh --verbose

# 查看失败详情
cat test_reports/test_results.md
```
