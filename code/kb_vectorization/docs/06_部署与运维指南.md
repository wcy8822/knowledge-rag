## 5. 自动启动配置

### 5.1 使用 launchd（macOS）

创建 `~/Library/LaunchAgents/com.knowledge-vectorization.plist`：

```xml
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
    <key>Label</key>
    <string>com.knowledge-vectorization</string>

    <key>ProgramArguments</key>
    <array>
        <string>/Users/didi/Downloads/panth/kb_vectorization/scripts/start_api.sh</string>
        <string>--daemon</string>
    </array>

    <key>WorkingDirectory</key>
    <string>/Users/didi/Downloads/panth/kb_vectorization</string>

    <key>RunAtLoad</key>
    <true/>

    <key>KeepAlive</key>
    <true/>

    <key>StandardOutPath</key>
    <string>/Users/didi/Downloads/panth/kb_vectorization/logs/launchd.stdout</string>

    <key>StandardErrorPath</key>
    <string>/Users/didi/Downloads/panth/kb_vectorization/logs/launchd.stderr</string>
</dict>
</plist>
```

加载服务：

```bash
# 加载服务
launchctl load ~/Library/LaunchAgents/com.knowledge-vectorization.plist

# 启动服务
launchctl start com.knowledge-vectorization

# 停止服务
launchctl stop com.knowledge-vectorization

# 卸载服务
launchctl unload ~/Library/LaunchAgents/com.knowledge-vectorization.plist
```

### 5.2 使用 shell 脚本定时任务

## 6. 备份与恢复

### 6.1 备份策略

| 数据类型 | 备份频率 | 保留时间 |
|---------|---------|---------|
| 向量库 | 每日 | 7天 |
| 配置文件 | 每次修改 | 永久 |
| 日志 | 每日清理 | 30天 |
| 元数据 | 每次更新 | 7天 |

### 6.2 备份脚本

```bash
#!/bin/bash
# 备份脚本

BACKUP_DIR="/Users/didi/Downloads/panth/kb_vectorization/data/backup"
DATE=$(date +%Y%m%d_%H%M%S)

# 创建备份目录
mkdir -p "$BACKUP_DIR"

# 备份向量库
echo "备份向量库..."
tar -czf "$BACKUP_DIR/vector_db_$DATE.tar.gz" data/vector_db/

# 备份配置
echo "备份配置..."
cp config/config.yaml "$BACKUP_DIR/config_$DATE.yaml"

# 备份元数据
echo "备份元数据..."
cp data/vector_db/metadata.json "$BACKUP_DIR/metadata_$DATE.json"

# 清理旧备份（保留最近7天）
echo "清理旧备份..."
find "$BACKUP_DIR" -name "*.tar.gz" -mtime +7 -delete
find "$BACKUP_DIR" -name "config_*.yaml" -mtime +30 -delete
find "$BACKUP_DIR" -name "metadata_*.json" -mtime +30 -delete

echo "备份完成: $DATE"
```

```bash
#!/bin/bash
# 监控脚本

# 检查内存
MEMORY=$(ps aux | grep "api.server" | grep -v grep | awk '{sum+=$4} END {print sum}')
MEMORY_INT=${MEMORY%.*}

if [ "$MEMORY_INT" -gt 80 ]; then
    echo "警告: 内存使用过高: ${MEMORY}%"
    # 发送告警（可集成邮件/短信）
```

## 9. 故障处理

### 9.1 常见故障

| 故障 | 症状 | 解决方法 |
|------|------|---------|
| 服务无法启动 | 端口冲突 | 检查端口占用，更换端口 |
| 内存不足 | 进程被杀 | 减小批次大小 |
| 向量化失败 | 文件损坏 | 检查文件编码 |
| API 响应慢 | 磁盘 I/O 慢 | 检查磁盘状态 |

### 9.2 故障排查流程

```
1. 检查日志
   ↓
2. 检查系统状态
   ↓
3. 检查配置
   ↓
4. 重启服务
   ↓
5. 如未解决，回滚
```
