> 核心理念：不是 AI 调用数据，而是数据主动赋能 AI
> 所有本地数据通过一个统一的 MCP Server 暴露给任何大模型

## 三、9 个 MCP Tool 设计

### 第一层：搜索与问答（已有）
1. **search_knowledge** — 文档语义搜索（ChromaDB + BGE-M3）
2. **ask_knowledge** — RAG 问答（搜索 + 本地 LLM 总结）

### 第二层：结构化数据（核心新增）
3. **query_mysql** — 自然语言转 SQL 查询
   - 输入："商户画像表里 KA 站点有多少"
   - 过程：理解表结构 → 生成 SQL → 执行 → 返回结果
   - 安全：只允许 SELECT，C3/C4 字段自动脱敏
4. **explain_sql_pipeline** — 解释 SQL 管道
   - 输入："overlap 管道的 Step 4 做了什么"
   - 过程：读 SQL 文件 → 解析逻辑 → 中文解释

### 第三层：笔记与上下文
5. **read_notes** — 读 Obsidian 笔记
   - 按路径/标签/日期/项目查询
   - 返回笔记内容 + frontmatter 标签
6. **project_context** — 获取项目上下文
   - 输入项目名 → 返回 WORK-LOG + README + 最近 git 变更
   - AI 进入任何项目前自动调用

### 第四层：学习与写入
7. **learn** — AI 主动写入新知识
   - AI 在对话中发现了新的业务理解 → 写入知识库
   - 自动更新 ChromaDB + Obsidian 笔记
8. **add_worklog** — 写工作日志
   - 每轮对话结束 → 自动记录核心决策到项目 WORK-LOG
9. **git_history** — 查代码变更历史
   - 输入："tag_ct_clean 最近改了什么"
   - 返回最近 commit + diff 摘要

## 四、关键创新点

### 4.1 上下文自动注入
AI 不需要主动调用 tool——MCP server 在每次对话开始时，根据当前工作目录自动注入：
- 项目 WORK-LOG 的最近 5 条记录
- 相关的知识库摘要（按项目名搜索）
- MySQL 表结构概览

### 4.2 知识回流闭环
```
用户提问 → AI 搜索知识库 → 回答
                ↓
         发现新知识/纠正旧知识
                ↓
         调用 learn tool 写回知识库
                ↓
         下次提问时知识已更新
```

### 4.3 SQL 安全查询
- 自动识别 C3/C4 字段（手机号、身份证、银行卡等）
- 查询结果自动脱敏后返回
- 只允许 SELECT，禁止 INSERT/UPDATE/DELETE
- 查询日志记录到审计表

### 4.4 多源交叉推理
AI 可以同时：
- 搜文档了解业务逻辑
- 查 MySQL 看实际数据
- 读笔记看历史决策
- 看 git 了解代码变化

这样回答的质量远超单一数据源。

## 六、与 OpenClaw 的集成

OpenClaw（小龙虾）可以配置 MCP 连接到同一个 server：
- 用户在飞书/D-Chat 问业务问题
- OpenClaw 调用 unified-data-layer 搜索知识库 + 查 MySQL
- 返回有数据支撑的回答

这样任何 AI 工具都成为了你本地数据的智能前端。
